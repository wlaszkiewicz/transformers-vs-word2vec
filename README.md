# Transformers vs Word2Vec

This project demonstrates the fundamental differences in how sequential data is represented by **static word embeddings** (Word2Vec) versus **context-aware embeddings** generated by Transformer architectures (BERT). Developed as a challenge for the Artificial Intelligence 1 course at Wrocław University of Science and Technology.


The central illustration is polysemy, using the word *mole*, which has distinct meanings in biology, chemistry, and dermatology. Word2Vec collapses all senses into one vector; BERT separates them by context. The notebook also includes a negative example that exposes a real architectural limitation of BERT: its hard 512-token sequence length constraint.

## Setup 

```bash
pip install -r requirements.txt
```

The first run will download two pretrained models automatically:

- **word2vec-google-news-300** (~1.6 GB) via `gensim.downloader`
- **bert-base-uncased** (~440 MB) via HuggingFace `transformers`

Both downloads happen once and are cached locally for subsequent runs.

## Notebook Contents

| Section | Description |
|---|---|
| **Word2Vec** | Model loading, cosine similarity table, `most_similar` analysis, PCA visualizations |
| **Transformer Encoder (BERT)** | Contextual embedding extraction, cosine similarity by sense, PCA visualizations |
| **Comparison** | Side-by-side similarity table and interactive 3D PCA plots |
| **Negative Example** | BERT's >512-token failure: context beyond the cutoff is silently discarded |
| **Try It Yourself** | Interactive widget — enter your own sentences and words to compare both models live |
