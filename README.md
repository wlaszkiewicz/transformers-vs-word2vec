# Transformers vs Word2Vec

This project demonstrates the fundamental differences in how sequential data is represented by **static word embeddings** (Word2Vec) versus **context-aware embeddings** generated by Transformer architectures (BERT).

The central illustration is polysemy — specifically the word *mole*, which has distinct meanings in biology, chemistry, and dermatology. Word2Vec collapses all senses into one vector; BERT separates them by context. The notebook also includes a negative example that exposes a real architectural limitation of BERT: its hard 512-token sequence length constraint.

## How to Run

```bash
jupyter notebook
```

Open `notebook.ipynb` and run all cells sequentially from top to bottom. The first run will download two pretrained models automatically:

- **word2vec-google-news-300** (~1.6 GB) via `gensim.downloader`

### Try It Yourself — widget guide

The interactive widget lets you compare Word2Vec and BERT on your own input:

1. Enter **Sentence 1** and the **target word** it contains (e.g., `The bank approved my loan.` / `bank`).
2. Enter **Sentence 2** and its **target word** (e.g., `We sat on the bank of the river.` / `bank`).
3. Click **Compare**.

The widget prints the **cosine similarity** under each model and renders an **interactive 3D PCA plot** with your words highlighted alongside the pre-computed embeddings.

### Negative Example — how it works

BERT's positional embedding table is fixed at **512 tokens**. Any input longer than this is silently truncated before the model runs.

The notebook demonstrates this with two long passages (>512 tokens) containing the word *mole*:

- **Passage A:** The animal-sense context appears *early* — within the 512-token window. BERT reads it and produces an embedding close to the animal-sense reference.
- **Passage B:** The chemistry-sense context appears *late* — beyond the cutoff. BERT never reads it. The resulting embedding drifts away from the chemistry-sense reference, because BERT only saw generic filler text.

The cosine similarity table and PCA plot quantify and visualize this drift.
